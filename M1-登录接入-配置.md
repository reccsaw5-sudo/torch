## Step 2（已完成，插件路线，不动核心）
auth.py 有 8000+ 行、被 17k 测试覆盖，且通用设备码函数写死了 nous 端点路径，直接改风险大。
改用 **插件路线**（AGENTS.md 推荐），核心零改动：

- `plugins/torch_login/`（通用插件）——注册 `hermes torch login/status/logout`
  - `casdoor_oauth.py`：授权码 + PKCE + loopback(53682) 收 code → 换 token → `/api/userinfo` → 存 `<HERMES_HOME>/torch_auth.json`(0600)
  - loopback 端点对 `trust_env` 做处理：本机回环绕过系统代理（否则 macOS 系统代理会 502），真实域名仍尊重代理
- `plugins/model-providers/torch/`——品牌 provider profile，`base_url=TORCH_INFERENCE_BASE_URL`（默认 `https://api.torchai.cn/v1`，M2 换成计量代理）

启用：`hermes plugins enable torch_login`（已启用）。

已验证：
- `hermes torch login` 生成正确 PKCE 授权 URL + 起 loopback 监听（等浏览器回调）
- 端到端换 token（脚本驱动 Casdoor 出 code）：拿到 access_token(JWT)/refresh_token/id_token，7 天有效；userinfo=admin
- `hermes torch status` 显示已登录账号
- `scripts/run_tests.sh` 插件相关 112 项通过；ruff 通过

注意：该 Casdoor 应用为 confidential client（换 token 需 client_secret）。分发客户端可：
(a) 配 `TORCH_CLIENT_SECRET`（secret 可被提取，属已知取舍）；或
(b) M2 把"换 token"改到业务后台代做（后台持 secret）。当前默认走 (a) 联调。

环境变量（可覆盖，不改代码）：
`TORCH_CASDOOR_ENDPOINT` / `TORCH_CLIENT_ID` / `TORCH_CLIENT_SECRET` / `TORCH_REDIRECT_PORT` / `TORCH_SCOPE` / `TORCH_TRUST_ENV` / `TORCH_INFERENCE_BASE_URL`。
